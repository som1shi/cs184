Title, Summary, and Team Members

Terra: A Procedural World Renderer

Terra is a real-time, browser-based renderer that integrates three physically-based systems: procedural terrain shaped by GPU-accelerated hydraulic erosion, an FFT ocean simulation using the Tessendorf wave model, and precomputed atmospheric scattering following Bruneton & Neyret (2008). All three share a sun direction and heightmap, so a time-of-day slider produces consistent changes across the sky, water, and terrain in the same scene. The demo runs entirely in WebGPU inside Chrome with no installation required.

Team members: Sarvagya Somvanshi, Mohan Xu, Afreen Shameem, Owen Ye

Problem Description

Rendering a convincing outdoor scene requires three physically distinct systems — terrain, water, and atmosphere — to share consistent physical parameters. Each system individually is a solved problem with published papers and reference implementations. The actual challenge is coupling them: the terrain heightmap defines the ocean waterline, the sun direction drives both wave reflections and sky color, and aerial perspective requires the atmosphere to operate on the terrain geometry. Getting these interactions correct in a single real-time WebGPU renderer is the core goal of this project.

Each subsystem has a specific technical bottleneck. Hydraulic erosion requires updating water flow, sediment transport, and deposition across every heightmap cell every timestep — a data-parallel problem that maps directly to GPU compute shaders but is impractical on the CPU. Ocean rendering requires summing thousands of wave components per frame; the inverse Fast Fourier Transform is the standard solution and the only approach that scales to real-time. Accurate atmospheric scattering requires integrating multiple Rayleigh and Mie scattering events along every view ray, which is solved by precomputing the scattering integrals into 4D lookup textures at startup and sampling them in constant time during rendering.

All three subsystems will be implemented as WebGPU compute and render passes in TypeScript and WGSL, targeting Chrome on Apple Silicon Macs. Everything runs client-side with no server or cloud compute required.

Goals and Deliverables

What we plan to deliver (baseline)

These are the things we need to get right to consider the project a success:

  • Eroded terrain: A procedurally generated island heightmap built from fractional Brownian motion noise, then eroded using a GPU implementation of the Mei et al. (2007) shallow-water pipe model. The simulation tracks terrain height, water height, suspended sediment, outflow flux in four directions, and velocity at every grid cell — all stored as GPU textures updated in parallel compute passes. The result should show realistic river valleys, ridges, and sediment deposits that you wouldn't get from unprocessed noise.
  • FFT ocean: A real-time ocean surrounding the island, using Tessendorf's approach where the surface height is decomposed into wave components with amplitudes drawn from the Phillips spectrum and then reconstructed via inverse FFT every frame — the same technique used in Sea of Thieves and Assassin's Creed IV. The water material will use Fresnel reflectance and depth-based color extinction.
  • Precomputed atmospheric scattering: A physically accurate sky using Bruneton & Neyret (2008). The scattering data is precomputed for all viewpoints, view directions, and sun directions at startup, then looked up in constant time at runtime. This gives correct blue sky during the day, golden-hour colors at sunset, and aerial perspective that hazes out distant terrain.
  • Time-of-day slider: A UI control that moves the sun from sunrise through sunset, with sky color, water reflections, and terrain shadows all responding to the same shared sun direction uniform.
  • Flythrough camera: Basic mouse/keyboard controls to navigate around the scene.
  • Browser demo: Everything delivered as a single browser tab (Chrome + WebGPU), shareable via URL with no install.

What we hope to deliver (aspirational)

If we get ahead of schedule, we'd like to add:

  • Interactive rainfall: Clicking on the terrain places a water source that runs the erosion simulation live, visibly carving new channels in real time.
  • Foam and whitecaps: Driven by the Jacobian of the wave displacement field as described in Tessendorf's paper, so waves crest and foam where they're steepest.
  • Slope-based terrain texturing: Rock on steep faces, grass on shallower slopes, sand at the waterline, using the erosion flow map to inform material type.
  • Day/night cycle: Extending past sunset to a starfield sky.

How we'll measure success

  • Erosion: Side-by-side comparison of the heightmap before and after erosion, checking that river valleys, ridges, and sediment deposits look geologically plausible.
  • Ocean: Visual comparison against reference FFT ocean renders; wave choppiness and Phillips spectrum parameters tuned to match Beaufort scale descriptions.
  • Atmosphere: Comparison of sky gradient and aerial perspective against Bruneton's own published validation renders.
  • Performance: The full integrated scene should run at ≥30fps at 1080p on an M-series MacBook. We'll profile each subsystem individually using the WebGPU timestamp query API to find any bottlenecks.

Resources

Papers

  • Mei, X., Decaudin, P., Hu, B.-G. (2007). Fast Hydraulic Erosion Simulation and Visualization on GPU. 15th Pacific Conference on Computer Graphics and Applications. DOI: 10.1109/PG.2007.15.
  • Tessendorf, J. (2001). Simulating Ocean Water. SIGGRAPH Course Notes.
  • Bruneton, E., Neyret, F. (2008). Precomputed Atmospheric Scattering. Computer Graphics Forum 27(4). DOI: 10.1111/j.1467-8659.2008.01245.x.

Reference implementations (open source)

  • WebGPU terrain erosion: GPU-Gang/WebGPU-Erosion-Simulation — TypeScript/WGSL, implements the Mei et al. pipe model
  • WebGPU FFT ocean: Popov72/OceanDemo — BabylonJS + WebGPU, Tessendorf model
  • Bruneton WebGL2 demo: ebruneton/precomputed_atmospheric_scattering — the original author's WebGL2 port (we'll need to translate GLSL to WGSL)
  • Bruneton WebGPU/WGSL port: TerrainView7 — independent WGSL implementation with compute shader precomputation

Platform and tooling

  • Language: TypeScript
  • Graphics API: WebGPU (Chrome 113+ on macOS via Metal backend)
  • Shader language: WGSL
  • Build tooling: Vite or Next.js
  • Hardware: Apple Silicon MacBooks (M1/M2/M3) — all team members have these, and all simulation runs client-side in the browser with no cloud compute required
