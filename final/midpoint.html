Progress Summary

Terra is a fully browser-based 3D procedural world engine built on WebGPU with zero runtime dependencies. Over the past two weeks we implemented three core subsystems: a procedural terrain with GPU-driven hydraulic erosion and vegetation, an FFT ocean simulation with Fresnel reflections, and atmospheric scattering based on Bruneton's atmospheric scattering. All three render together in a unified scene.

Preliminary Results

Terrain

A 512×512 heightmap is generated on the CPU using 7-octave fBm Perlin noise (lacunarity 2.0, persistence 0.40) with domain warping to break grid alignment, then shaped by a radial island falloff and Gaussian blur. The heightmap is then run through a GPU erosion pipeline consisting of 12 compute shader entry points orchestrated across 300 outer iterations with thousands of dispatches. Erosion follows the Stream Power Equation. The terrain fragment shader blends five biomes via triplanar texturing driven by altitude and slope, with noise-perturbed thresholds for organic transitions. Lighting uses a three-light rig ray-marched soft shadows. Rivers are rendered by sampling flow accumulation data with Phong specular highlights.

Ocean

The ocean uses Tessendorf-style FFT wave synthesis driven by the Phillips spectrum: The spectrum is initialized via Box-Muller Gaussian sampling with conjugate symmetry for real-valued output. An in-place radix-2 Cooley-Tukey FFT on a 128×128 grid reconstructs the height field as a separable 2D IFFT, running on the CPU with per-frame texture upload. The ocean renders as a 256×256 mesh with Schlick Fresnel reflections.

Atmosphere & Sky

The sky renders as a full-screen quad behind all geometry with a time-of-day system driving morning, evening, and night transitions with sunset/sunrise coloring and seed-based hue variation. Procedural clouds use 3-octave value noise with time-of-day drift while a sun disk and moon complete the sky dome. Precomputed transmittance, scattering, and irradiance textures have been generated via compute shader using Bruneton's atmospheric parameters.

Current Status

We are on track with our original timeline. All primary rendering features targeted for Milestone 1 are implemented and functional. The main area we need to touch up is atmosphere integration as we need to finish connecting the precomputed sky data to the final rendering for the most accurate lighting and color. In addition, we are working to port ocean FFT to GPU instead of CPU. Outside of that, we'll do work on some of our stretch (Aspirational!) goals.
